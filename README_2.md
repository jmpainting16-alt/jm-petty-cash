# ระบบเบิกเงินสดย่อย — JM

เว็บแอปหน้าเดียว (ไฟล์ `index.html` ไฟล์เดียวจบ) สำหรับจัดการเงินสดย่อยของ
บริษัท เจเอ็ม. เพ้นท์ติ้งแอนด์ซัพพลาย จำกัด — บันทึกรายจ่าย หักยอดอัตโนมัติ
ออกใบสำคัญเบิกจ่าย/ใบสรุป ติดตามรอบเงินสดย่อย และการโอนคืนพนักงาน

## วิธีนำขึ้น GitHub Pages (ใช้งานจริงผ่านเว็บ)

1. เข้า https://github.com แล้วล็อกอิน → กดปุ่ม **New** สร้าง repository ใหม่
   (ตั้งชื่อเช่น `jm-petty-cash` เลือก **Public**)
2. ในหน้า repo กด **Add file → Upload files** แล้วลากไฟล์ **`index.html`** ขึ้นไป → **Commit changes**
3. ไปที่ **Settings → Pages**
4. ที่หัวข้อ **Build and deployment → Source** เลือก **Deploy from a branch**
   จากนั้นเลือก branch **main** โฟลเดอร์ **/ (root)** → **Save**
5. รอประมาณ 1–2 นาที หน้านั้นจะขึ้นลิงก์
   `https://<ชื่อผู้ใช้>.github.io/jm-petty-cash/` — เปิดลิงก์นี้ใช้งานได้เลย

### ใช้บนมือถือให้เหมือนแอป
เปิดลิงก์ด้านบนบนมือถือ → กดเมนูเบราว์เซอร์ → **เพิ่มไปยังหน้าจอโฮม / Add to Home Screen**

## รหัสอนุมัติ (Approval PIN)

ทุกคนเปิดแอปและเห็นข้อมูลได้เหมือนกันทั้งหมด แต่การกระทำสำคัญต้องใส่ "รหัสอนุมัติ" ก่อน ได้แก่
อนุมัติ/ปฏิเสธคำขอ, ปิดรอบเงินสดย่อย, ลบรายการ และล้างข้อมูล

ตั้งรหัสได้ที่ **ตั้งค่า → รหัสอนุมัติ** (เปลี่ยน/ลบรหัสต้องใส่รหัสเดิมก่อน)
มุมขวาบนจะมีป้ายบอกสถานะว่าตั้งรหัสแล้วหรือยัง

หมายเหตุ: รหัสนี้เป็นการล็อกใช้งานเบื้องต้นฝั่งเบราว์เซอร์ เหมาะกับการกันคนกดอนุมัติโดยไม่ได้รับอนุญาต

## การเก็บข้อมูล (สำคัญมาก)

- ข้อมูลถูกเก็บไว้ใน **เบราว์เซอร์/เครื่องที่ใช้งานเท่านั้น** (ผ่าน IndexedDB / localStorage)
  ไม่ได้ส่งขึ้นเซิร์ฟเวอร์ และ **ไม่ซิงก์ข้ามเครื่องอัตโนมัติ**
- ใช้เมนู **ตั้งค่า → สำรอง / กู้คืนข้อมูล**
  - **สำรองข้อมูล** = ดาวน์โหลดไฟล์ `.json` (รวมรูปทั้งหมด) เก็บไว้กันข้อมูลหาย
  - **กู้คืนจากไฟล์** = นำไฟล์สำรองกลับเข้ามา หรือใช้ย้ายข้อมูลไปอีกเครื่อง
    เช่น ส่งไฟล์ให้ CEO เปิดดูบนเครื่องตัวเอง
- ถ้าล้างข้อมูลเบราว์เซอร์/ถอนแอป ข้อมูลจะหาย — **ควรกดสำรองเป็นประจำ**

## ข้อจำกัด & ก้าวต่อไป

GitHub Pages เป็นโฮสต์แบบสแตติก (ไม่มีฐานข้อมูลกลาง) จึงเหมาะกับการใช้งาน
บนเครื่องหลักของผู้ถือเงินสดย่อย แล้วส่งไฟล์สำรองให้ CEO ดู

ถ้าต้องการให้ **พนักงานหลายคนส่งเบิกจากมือถือตัวเอง และ CEO เห็นข้อมูลชุดเดียวกันแบบเรียลไทม์**
จะต้องต่อกับฐานข้อมูลกลาง (เช่น Firebase, Supabase หรือ Google Sheets) — ทำเพิ่มได้
